6.092 Assignment 7: Refining YourFace

Your assignment is to use your current knowledge about packages, Collections, and abstract classes to refine your code from Assignment 6.

Part 1: Reinforcing the Abstraction Barrier

Add and incorporate these abstract classes into your code.

    AbstractSocialEntity implements SocialEntity
  • a constructor
  • name : String
  • getName() : String
  • id : long
  • getId() : long
  • abstract toString() : String
    AbstractSocialGrouping extends AbstractSocialEntity
  • a constructor
  • members : Set<Person>
  • getMembers() : Set<Person>
  • addMember(Person)
  • removeMember(Person) optional

Now that you have these abstract classes, you must extend them in the appropriate classes from Assignment 6, instead of implementing SocialEntity directly. There is no need to rewrite the SocialEntity interface, but you will need to rewrite the constructors and some of the relevant methods for each of Person, Group, Network, and Band. Be mindful of how and when you initialize or assign each class its fields. Remember to use the super constructor, now that you're extending abstract classes. Also remember to use protected keyword on the constructors in the above abstract classes.

Part 2: Improving Functionality

Write a Relationship class and create one or more instances; incorporate this into the Person class to model a person's primary relationship with another person. Also incorporate Collections into Person, YourFace, and the other appropriate classes; there should no longer be any need for arrays.

    Relationship class
  • a constructor
  • type : String
  • isMutual : boolean
  • toString() : String
E.g.: friend, enemy, coworker, classmate, sibling, and roommate vs. teacher, student, brother, parent, fan, etc. The Relationship class should not extend anything.
    Person extends AbstractSocialEntity
  • a (modified) constructor
  • acquaintances : Map<Person, Relationship>
  • networks : Set<Network>
  • location : String
  • addAcquaintance(Person, Relationship) (optional)
  • toString() : String
    Optional alternative: Person extends AbstractSocialEntity
  • a (modified) constructor
  • acquaintances : Map<Person,Set<Relationship>>
  • networks : Set<Network>
  • location : String
  • addAcquaintance(Person)
  • addAcquaintance(Person, Relationship)
  • toString() : String

Feel free to add additional methods or fields if they're necessary or useful. The optional alternative Person class would allow us to model multiple relationships between this and another Person, instead of only one "primary" relationship. NOTE: You may want to look into the Map methods put(K,V), get(K), keySet(), and entrySet(). Also optional: When adding a <Person, Relationship> mapping to a Person, you may also want to check whether or not the Relationship is mutual and take the appropriate action (i.e., add this Person and the same Relationship to said acquaintance's own acquaintance set).
